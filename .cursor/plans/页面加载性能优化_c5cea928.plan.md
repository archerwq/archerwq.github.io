# 页面加载性能优化计划

**本期范围**：执行第 1、2、3、4、5 项；第 6 项（移除 jQuery）留待后续。

---

## 1. 图片懒加载（LCP 与请求数）

**目标**：首屏外图片延迟加载，首屏内保持立即加载以保障 LCP。


| 文件                                                           | 改动                                                                                                                                 |
| ------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------- |
| [_layouts/post.html](_layouts/post.html)                     | 文章头图（首屏内）保持无 `loading` 或显式 `loading="eager"`；上下篇导航缩略图改为使用 `site.img_path` 并加 `loading="lazy"`（当前为 `/images/posts/`，与头图来源不一致易 404）。 |
| [index.html](index.html)                                     | 首页列表：第 1 条文章头图不加 lazy；第 2、3 条头图加 `loading="lazy"`（用 `forloop.index` 判断）。                                                           |
| [_includes/image_caption.html](_includes/image_caption.html) | `<img>` 增加 `loading="lazy"`。                                                                                                       |
| [_includes/image_full.html](_includes/image_full.html)       | `<img>` 增加 `loading="lazy"`。                                                                                                       |


**说明**：正文内图片多为首屏外，统一 lazy；若个别文章首图在首屏且来自 include，可后续再按需改为 eager。

---

## 2. Disqus 按需加载

**目标**：文章页不自动加载 Disqus，用户点击「加载评论」后再加载，减轻 TTI 与第三方请求。


| 文件                                             | 改动                                                                                                                                                                     |
| ---------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [_includes/disqus.html](_includes/disqus.html) | 去掉立即执行的 embed 脚本。改为：保留 `#disqus_thread` 容器；在其上方增加「加载评论」按钮；点击后注入现有 Disqus 脚本逻辑（`disqus_shortname`、创建 script 指向 embed.js、appendChild），并隐藏按钮、显示加载中或容器。保留 `<noscript>` 文案。 |


**预期**：文章页首次加载不再请求 `disqus.com/embed.js`，仅点击后加载。

---

## 3. 预连接与资源提示修正

**目标**：头图域名与预连接一致，并补充 jQuery CDN 的 dns-prefetch。


| 文件                                             | 改动                                                                                                                                                                                                                                       |
| ---------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [_layouts/default.html](_layouts/default.html) | 在 `<head>` 的 Resource Hints 中：增加 `preconnect` 与 `dns-prefetch` 到 `https://pic.imgdb.cn`（头图实际域名，与 `_config.yml` 中 `img_path` 一致）；保留对 `pic1.imgdb.cn` 的 preconnect（作者头像等）；增加 `dns-prefetch` 到 `https://code.jquery.com`（若后续移除 jQuery 可再删）。 |

## 6. 可选：移除 jQuery 依赖（中优先级）

**目标**：去掉约 90KB 的 jQuery，用原生 JS 实现菜单与回到顶部；FitVids 仅在有视频的页面按需加载。


| 文件                                             | 改动                                                                                                                                                   |
| ---------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| [_layouts/default.html](_layouts/default.html) | 移除 jQuery 的 `<script defer src="https://code.jquery.com/...">`；保留或改为内联/单独文件的 `plugins.js`（FitVids）仅在有 `.fluid-width-video-wrapper` 或 iframe 时加载（可选）。 |
| [js/custom.js](js/custom.js)                   | 用原生 JS 重写：`#menu-toggle` 的 click、aria-expanded、`.nav-menu` 的显示/隐藏（如 classList + CSS transition 或 slide）；`#top-link` 的 scrollTo；移除对 jQuery 的依赖。       |
| [js/plugins.js](js/plugins.js)                 | 保留 FitVids 逻辑；若需按需加载，可在 custom.js 中检测文章内是否有视频 iframe，有则动态插入 plugins.js。                                                                              |


本项放在本期之后再做。

## 测试方案

**环境**：本地 `jekyll serve` 与 GitHub Pages 构建结果（或 `bundle exec jekyll build` 后检查 `_site`）。

1. **功能回归**
  - 首页：分页、每篇文章标题/头图/链接正确；第 2、3 条头图滚动到视口内时才加载（DevTools Network 中过滤 Img，观察 lazy 请求时机）。
  - 文章页：头图、正文、作者信息、上下篇导航（含缩略图）正常；点击「加载评论」后 Disqus 出现且评论可正常使用。
  - 关于页、标签页、侧栏最新文章与标签链接正常；移动端菜单展开/收起、回到顶部正常。
2. **性能（Chrome DevTools）**
  - **Lighthouse**（Performance）：在优化前后分别跑 3 次取中位数，对比 FCP、LCP、TTI、TBT；关注「Defer offscreen images」与「Reduce the impact of third-party code」是否改善。
  - **Network**：Throttling 设为 Fast 3G，刷新文章页，确认首屏请求中无 `disqus.com/embed.js`；点击加载评论后再出现；首屏图片仅首图（及作者头像等）立即加载，其余 lazy。
3. **兼容与降级**
  - 关闭 JS：Disqus 显示 noscript 文案。
  - 旧浏览器：`loading="lazy"` 会被忽略，图片仍会加载，仅无懒加载效果。
4. **清单**
  - 首页 3 条列表图：第 1 条 eager，第 2、3 条 lazy。
  - 文章页头图不 lazy；正文内图与 include 图 lazy。
  - 文章页 Disqus 仅点击后加载且功能正常。
  - preconnect/dns-prefetch 包含 pic.imgdb.cn 与 code.jquery.com（或移除 jQuery 后删除后者）。
  - 侧栏最新文章数量为 10（或 15）。
  - 上下篇缩略图使用 img_path、无 404。
